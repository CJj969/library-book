## D. 建议优化项

### O1：允许预约"今日已过时段"的边界情况

**位置：** 第 2.5 节第 174 行 `reserveDate <= DateTime.Today.AddDays(-1)`。

**问题：**  
当前校验只拒绝"昨天的日期及更早"，今天的任何时段（包括已过的"上午"）都可以提交预约。虽然冲突检测会阻止已经被别人约走的时段，但如果一个座位今天"上午"时段空闲但现在已经下午 3 点，用户仍然可以提交"上午"的预约——这个预约永远不会被使用，还会在"我的预约"中显示为"已预约"直到第二天才变成"已完成"。

**建议：** 在 `CreateReservation()` 中增加对当日已过时段的拦截（与取消时复用同一套 `IsTimeSlotPast` 逻辑）：

```csharp
// 在日期校验之后增加：
if (reserveDate == DateTime.Today && IsTimeSlotPast(timeSlot))
    return (false, "该时段已过，请选择其他时段");
```

如果选择不改，也建议在文档中注明"当前设计允许预约当日已过时段"作为已知行为。

### O2：时段阈值集中管理

**位置：** 第 2.7 节第 289~299 行 `IsTimeSlotPast()` 方法。

三个时段的截止时间（12:00 / 17:00 / 22:00）目前硬编码在 Service 私有方法中。第 4.3 节已提到"建议提取为 `DateTimeHelper` 静态方法"，建议顺带把时段阈值也提取为 `static readonly Dictionary<string, int>`，方便两处 Service（Reservation + Statistics）统一引用。
