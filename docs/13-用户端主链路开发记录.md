## 核心实现说明

### 用户身份管理（Session）

- `POST /Reservation/Switch`：从 Seed 用户列表切换身份，写入 Session[UserId/UserName/UserRole]
- `POST /Reservation/Logout`：清除 Session
- Layout 顶部显示当前用户名；未登录显示"体验账号"
- 需要登录的 Action（MyReservations / Create / Cancel）自动检查 Session

### 座位列表

- 区域 Tab 切换（从 Area 字段自动提取去重列表）
- 每个座位卡片显示编号 + 描述 + 状态 Badge（空闲=绿色/已预约=灰色/维修中=灰色）
- 维修中与已预约的座位点击无跳转（disabled-link 样式）

### 座位详情

- 上方：座位信息卡（区域 + 描述 + 可用状态）
- 中间：今日 3 时段时间轴（绿=空闲 / 红=已预约 / 灰=已过）
- 下方：预约表单（日期选择器 min=today + 时段下拉），仅已登录用户可见

### 预约提交流程

1. 验证座位存在 + 启用状态
2. 日期校验（不可约过去日期）
3. 时段校验（不可约已过时段）
4. 冲突检测（Service 层查询 + DB 唯一索引双保险）
5. 写入 Reservation 表 → 跳转到"我的预约"
6. 失败时返回详情页并显示错误信息（TempData）

### 我的预约

- 按创建时间倒序排列
- 状态逻辑：已取消/已预约/已完成（过期自动判定，不修改 DB）
- "已预约"状态显示"取消预约"按钮，带 `confirm()` 弹窗

### 取消预约

- 归属校验（只能取消自己的）
- 状态校验（只能取消"已预约"的）
- 时间校验（已过期或已过时段不可取消）
- 成功后 Status → "已取消"
