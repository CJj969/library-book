## 4. 未接入逻辑

以下逻辑在 static-v1 中未实现，需要在后续开发中接入后端：

| 逻辑 | 说明 |
|------|------|
| 账号切换 | 导航栏下拉框为静态 HTML，切换不改变页面内容 |
| 预约提交 | 表单提交指向 `#`，无后端校验和数据库写入 |
| 预约冲突校验 | 错误提示为静态展示，非实时校验 |
| 取消预约 | `confirm()` 弹窗仅改变 DOM 文本，不持久化 |
| 管理员登录 | 表单无后端验证，直接跳转 |
| 座位 CRUD | 新增/编辑/删除为静态弹窗示意，无真实数据操作 |
| 统计数据 | 数字为硬编码，非数据库聚合查询 |
| 区域筛选 | Tab 切换为静态交互示意，不过滤座位列表 |
| 过期自动完成 | 预约状态不会自动变更，需后端定时逻辑 |

## 5. 界面风格说明

### 用户端（手机优先）

- 使用 `.phone-frame` 容器模拟手机视图（最大宽度 480px，桌面端居中带阴影）
- 导航栏固定顶部 56px，包含品牌名、导航链接、账号切换下拉框
- 卡片式设计，统一 12px 圆角，浅色边框 `#E2E8F0`
- 内容区左右留边距 16px，底部安全区 24px
- 颜色以白色为底，蓝色 `#2563EB` 为主色点缀，绿色/红色/琥珀色为状态色

### 管理端（桌面优先）

- 全宽布局，内容区居中最大宽度 1200px
- 导航栏样式与用户端一致（白色背景，底部边框），链接不同
- 表格使用 Bootstrap 风格 + 自定义奇偶行色和 hover 效果
- 筛选栏、操作按钮、状态标签统一风格
- 统计卡片四列网格

### 共享组件

所有组件样式集中在 `css/style.css` 中，包括：

| 组件 | CSS 类前缀 | 位置 |
|------|-----------|------|
| 按钮 | `.btn-seat-*` | style.css 按钮系统 |
| 卡片 | `.card-seat` | style.css 卡片组件 |
| 表格 | `.table-seat` | style.css 管理端表格 |
| 状态标签 | `.status-tag.*` | style.css 状态标签 |
| 输入框 | `.form-control-seat` | style.css 输入框 |
| 提示条 | `.alert-seat-*` | style.css 提示条 |
| 空状态 | `.empty-state` | style.css 空状态 |
| 弹窗 | `.modal-*` | style.css 模态弹窗 |
| 统计卡片 | `.stat-card` / `.stat-card-admin` | style.css |
| 导航 | `.mobile-nav` / `.admin-nav` | style.css |
