# 图书馆座位预约系统 —— 系统设计说明

## 1. 系统设计目标

| 目标 | 说明 |
|------|------|
| 可运行 | `dotnet run` 后能在浏览器完整走通预约主流程 |
| 可演示 | Seed 数据预填充，开箱即展示，无需手动插数据 |
| 可维护 | 三层分离（Controller / Service / DataAccess），改动不影响全局 |
| 可交接 | 代码结构清晰，其他同学接手不需要逐文件问人 |

**设计原则：** 简单优先，不引入本项目不需要的抽象（如 Repository 泛型基类、Unit of Work、AutoMapper、MediatR）。EF Core 直查够用，就不要再包一层。

## 2. 技术栈说明

| 技术 | 版本 | 用途 |
|------|------|------|
| ASP.NET Core MVC | .NET 8 | Web 框架，路由 + Controller + Razor View |
| Razor | .NET 8 | 服务端页面模板 |
| Entity Framework Core | 8.x | ORM，数据库访问 |
| SQL Server LocalDB | 随 VS/SSDT 安装 | 本地开发数据库 |
| Bootstrap | 5.x | 前端 UI 框架 |
| 自定义 CSS | — | 覆盖 Bootstrap 默认样式（色值/圆角/间距等见 UI 规范） |

**不需要引入的依赖项：**
- Identity（本期不做真实账号体系）
- AutoMapper（实体和 ViewModel 手动映射即可）
- MediatR / FluentValidation（规模太小，Controller 内校验够用）
-任何 NuGet 包如果不是必须用就不加

## 4. 分层说明

### 4.1 三层职责

```
┌─────────────┐
│  Controller  │  ← 接收请求、调用 Service、返回 View 或 JSON
├─────────────┤
│  Service     │  ← 业务逻辑、校验、跨实体操作、聚合查询
├─────────────┤
│  DataAccess  │  ← EF Core DbContext、Entity 定义、Seed Data
└─────────────┘
```

### 4.2 Controller 层

**职责范围：**
- 解析路由参数和 QueryString
- 调用 Service 获取数据
- 将 Service 返回的数据组装为 ViewModel
- 返回 `View(viewModel)` 或 `RedirectToAction()`
- 处理表单提交的 ModelState 校验
- 设置/读取 Session

**禁止做的事：**
- ❌ 在 Controller 里直接操作 `_context`（DbContext）
- ❌ 在 Controller 里写 SQL 或 LINQ 查询
- ❌ 在 Controller 里写业务判断（如"是否冲突"、"是否能取消"）

**示例（HomeController）：**

```csharp
public class HomeController : Controller
{
    private readonly IStatisticsService _statisticsService;

    public HomeController(IStatisticsService statisticsService)
    {
        _statisticsService = statisticsService;
    }

    public IActionResult Index()
    {
        var stats = _statisticsService.GetHomePageStats();
        var vm = new HomeViewModel
        {
            TotalSeats = stats.TotalSeats,
            AvailableSeats = stats.AvailableSeats,
            ReservedToday = stats.ReservedToday
        };
        return View(vm);
    }
}
```

### 4.3 Service 层

**职责范围：**
- 通过构造函数注入 `AppDbContext`
- 执行业务逻辑（预约冲突检测、取消条件判断、统计计算）
- 调用 EF Core 进行增删改查
- 返回 DTO 或直接在 Service 中组装好 ViewModel 需要的数据

**命名规约：**
- 方法名以业务动作命名，不写 CRUD 泛名
  - ✅ `GetAvailableSeatsByArea(string area)`
  - ✅ `CreateReservation(int userId, int seatId, DateTime date, string period)`
  - ❌ `Save()` / `Update()` / `Delete()`

**示例（ReservationService）：**

```csharp
public class ReservationService : IReservationService
{
    private readonly AppDbContext _context;

    public ReservationService(AppDbContext context)
    {
        _context = context;
    }

    public (bool Success, string Message) CreateReservation(
        int userId, int seatId, DateTime date, string period)
    {
        // 1. 查座位是否存在
        var seat = _context.Seats.Find(seatId);
        if (seat == null) return (false, "座位不存在");

        // 2. 查时段是否冲突
        var conflict = _context.Reservations.Any(r =>
            r.SeatId == seatId &&
            r.Date == date &&
            r.Period == period &&
            r.Status == ReservationStatus.Active);
        if (conflict) return (false, "该时段已被预约");

        // 3. 创建预约
        var reservation = new Reservation
        {
            UserId = userId,
            SeatId = seatId,
            Date = date,
            Period = period,
            Status = ReservationStatus.Active,
            CreatedAt = DateTime.Now
        };
        _context.Reservations.Add(reservation);
        _context.SaveChanges();
        return (true, "预约成功");
    }
}
```

### 4.4 DataAccess 层

**职责范围：**
- `AppDbContext`：定义 DbSet、配置实体关系、设置字段约束
- `SeedData`：初始化体验账号、管理员账号、演示座位
- Entity 类定义（放在 `Models/Entities/` 下）

**不使用独立的 Repository 层**的原因：
- 本项目只有 3 张表，Service 直接依赖 `AppDbContext` 足够
- 多套一层 Repository 不会带来任何好处，只会增加文件数量
- 如果后期需要做缓存或数据源切换，再重构也来得及

### 4.5 ViewModel

**职责范围：**
- 承载一个页面需要的全部展示数据
- 包含字段标注（`[Required]`、`[Display(Name = "...")]`）
- 不包含业务逻辑

**规约：**
- 一个页面一个 ViewModel，不跨页面重用（避免"改 A 影响 B"）
- ViewModel 中只放展示需要的数据类型（`string` / `int` / `bool` / `SelectList`），不放 Entity 对象

### 4.6 Entity

**职责范围：**
- 数据库表的 C# 映射
- 包含数据注解（`[Key]`、`[Required]`、`[MaxLength]`）
- 包含导航属性（`public Seat Seat { get; set; }`）

**规约：**
- Entity 不包含业务逻辑方法
- Entity 不包含 UI 展示逻辑（如状态颜色、格式化的时间字符串）

## 7. 主链路工程落点

### 7.1 预约主链路（学生）

```
步骤                 Controller          Service 方法                          关键校验
─────────────────────────────────────────────────────────────────────────────────────────
① 访问首页           Home.Index          StatisticsService.GetHomePageStats()  无
② 查看座位列表       Seat.Index          SeatService.GetAllSeatsWithStatus()   无
③ 查看座位详情       Seat.Detail         SeatService.GetSeatDetail()           座位存在性
④ 提交预约(表单POST) Reservation.Create  ReservationService.CreateReservation() 座位存在/时段冲突
⑤ 查看我的预约       Reservation.My-    ReservationService.GetMyReservations() 当前用户身份
                     Reservations
⑥ 取消预约(POST)     Reservation.Cancel  ReservationService.CancelReservation() 状态=Active / 归属校验
```

**事务边界：** 步骤④ 中 `CreateReservation` 的写入操作在单次 `SaveChanges()` 内完成，EF Core 默认带隐式事务，不需要额外包装。

**POST 流程统一模式：**

```
用户提交表单 → Controller 校验 ModelState
  ├─ 不通过 → 返回 View(model)，显示 ValidationSummary
  └─ 通过 → 调用 Service
       ├─ Service 返回 (false, errorMsg) → TempData["Error"] → 重定向回来源页
       └─ Service 返回 (true, successMsg) → TempData["Success"] → 重定向到目标页
```

```
步骤                 Controller          Service 方法
```

## 8. 异常处理原则

### 8.1 分层处理策略

| 层 | 处理方式 |
|---|----------|
| Controller | 捕获 Service 返回的 `(bool, string)` 结果，写入 `TempData`，重定向或返回 View |
| Service | 不抛异常，返回元组 `(bool Success, string Message)` 作为标准结果格式 |
| DataAccess | 不处理，让 EF Core 异常自然上抛给 Service |

### 8.2 不使用的异常机制

- ❌ 不在 Service 中抛 `ArgumentException` / `InvalidOperationException` 等异常来表示业务拒绝
- ❌ 不写全局异常过滤器（`IExceptionFilter`）
- ❌ 不使用 `try-catch` 包裹正常业务流程

**理由：** 业务拒绝（如时段冲突、预约已过期）是预期内的交互结果，不是异常。用 return value 比用异常更清晰、更可控。

### 8.3 三类异常及处理

| 异常类型 | 表现 | 处理 |
|----------|------|------|
| 预期业务拒绝 | 表单提交后 TempData 红色提示 | Controller 根据 Service 返回值处理 |
| 数据不存在 | 详情页传入不存在 ID | Service 返回 `(false, "xxx不存在")` |
| 系统异常 | DB 连接失败、SQL 异常 | ASP.NET Core 默认开发者异常页（开发环境）/ 通用错误页（生产环境） |

### 8.4 TempData 使用规约

```csharp
// 成功
TempData["Success"] = "预约成功";
return RedirectToAction("MyReservations");

// 失败
TempData["Error"] = "该时段已被预约";
return RedirectToAction("Detail", new { id = seatId });
```

_Layout 或具体 View 中统一读取并渲染提示条：

```html
@if (TempData["Success"] != null)
{
    <div class="alert-seat alert-seat-success">@TempData["Success"]</div>
}
@if (TempData["Error"] != null)
{
    <div class="alert-seat alert-seat-danger">@TempData["Error"]</div>
}
```

## 9. Session / Cookie 使用原则

### 9.1 体验账号（学生端）

| 字段 | 存储位置 | 说明 |
|------|----------|------|
| 当前用户 ID | Session["UserId"] | 切换账号时更新 |
| 当前用户名 | Session["UserName"] | 导航栏显示用 |
| 当前用户角色 | Session["UserRole"] | "Student" / "Admin" |

**切换逻辑（导航栏下拉框 POST 到 `/Reservation/Switch`，放在 ReservationController 中）：**

```csharp
[HttpPost]
public IActionResult Switch(int userId)
{
    var user = _context.Users.Find(userId);
    if (user == null) return RedirectToAction("Index", "Home");

    HttpContext.Session.SetInt32("UserId", user.Id);
    HttpContext.Session.SetString("UserName", user.Name);
    HttpContext.Session.SetString("UserRole", user.Role);

    return RedirectToAction("Index", "Home");
}
```

### 9.2 管理员登录

| 字段 | 存储位置 | 说明 |
|------|----------|------|
| 管理员 ID | Session["AdminId"] | 登录成功时设置 |
| 管理员名 | Session["AdminName"] | 导航栏显示 |
| 角色 | Session["UserRole"] | "Admin" |

**登录校验：**

```csharp
public (bool Success, string Message) ValidateLogin(string username, string password)
{
    var admin = _context.Users.FirstOrDefault(u =>
        u.Name == username && u.Role == "Admin");

    if (admin == null) return (false, "账号或密码错误");
    if (admin.Password != password) return (false, "账号或密码错误");
    return (true, null);
}
```

### 9.3 Session 拦截配置

**用户端：** 不拦截（体验账号未切换时给予默认账号，或停留在首页）

**管理端：** 在 AdminController 中添加 `OnActionExecuting` 过滤器或使用基类：

```csharp
public class AdminBaseController : Controller
{
    public override void OnActionExecuting(ActionExecutingContext context)
    {
        var role = HttpContext.Session.GetString("UserRole");
        if (role != "Admin")
        {
            context.Result = RedirectToAction("Login");
        }
        base.OnActionExecuting(context);
    }
}
```

AdminController 继承 `AdminBaseController`，Login 和 Logout 除外（通过 `[AllowAnonymous]` 跳过）。

### 9.4 Cookie 配置

```csharp
// Program.cs
builder.Services.AddDistributedMemoryCache();
builder.Services.AddSession(options =>
{
    options.IdleTimeout = TimeSpan.FromMinutes(30);
    options.Cookie.HttpOnly = true;
    options.Cookie.IsEssential = true;
});
```
