# 图书馆座位预约系统 —— 关键链路详细设计

## 1. 关键链路选择说明

从前序文档中识别并确认两条主链路：

| 链路 | 名称 | 来源 | 优先级 |
|------|------|------|--------|
| 链路一 | 用户预约主链路（查座→预约→取消） | 页面树 §4、系统设计 §7.1 | P0（核心基础） |
| 链路二 | 管理端处理链路（登录→管理→统计） | 页面树 §5、系统设计 §7.2 | P1（主体业务） |

**范围约定：** 两条链路均已在前序文档中明确定义，本文件不做步骤改名或职责重分配。对每条链路的每个步骤展开到：路由 → Controller → Service → DataAccess → DB 变化 → 状态流转 → 异常边界。

## 2. 链路一：用户预约主链路

```csharp
public List<SeatItem> GetAllSeatsWithStatus(DateTime today)
{
    var seats = _context.Seats.ToList();
    var reservedSeatIds = _context.Reservations
        .Where(r => r.ReserveDate == today && r.Status == "已预约")
        .Select(r => r.SeatId)
        .Distinct()
        .ToHashSet();

    return seats.Select(s => new SeatItem
    {
        Id = s.Id,
        SeatNumber = s.SeatNumber,
        Area = s.Area,
        Description = s.Description,
        IsActive = s.IsActive,
        // 维修中的座位即使没有预约也不可约
        IsReservedToday = !s.IsActive || reservedSeatIds.Contains(s.Id)
    }).ToList();
}
```

### 2.5 步骤④：提交预约（关键步骤）

| 维度 | 内容 |
|------|------|
| 页面 | 座位详情页内嵌表单（PC-03） |
| 路由 | `POST /Reservation/Create` |
| Controller | `ReservationController.Create(CreateReservationViewModel model)` |
| 权限 | 需要体验账号（Session["UserId"] != null） |
| Service | `IReservationService.CreateReservation(int userId, int seatId, DateTime reserveDate, string timeSlot)` |
| DataAccess | ① `_context.Seats.FindAsync(seatId)` — 查座位 ② `_context.Reservations.AnyAsync(...)` — 冲突检测 ③ `_context.Reservations.Add()` + `SaveChangesAsync()` — 写入 |
| DB 变化 | INSERT 一条 Reservation 记录 |
| ViewModel | `CreateReservationViewModel { SeatId, ReserveDate, TimeSlot }` |
| 成功路径 | Service 返回 `(true, "预约成功")` → TempData["Success"] → `RedirectToAction("MyReservations")` |
| 失败路径 | Service 返回 `(false, 错误消息)` → TempData["Error"] → `RedirectToAction("Detail", new { id = seatId })` |

**Service 完整实现：**

```csharp
private static readonly string[] ValidTimeSlots = { "上午", "下午", "晚上" };
private static readonly Dictionary<string, int> TimeSlotEndHours = new()
{
    ["上午"] = 12,
    ["下午"] = 17,
    ["晚上"] = 22
};

public (bool Success, string Message) CreateReservation(
    int userId, int seatId, DateTime reserveDate, string timeSlot)
{
    // 1. 座位存在性校验
    var seat = _context.Seats.Find(seatId);
    if (seat == null)
        return (false, "座位不存在");

    // 2. 座位启用状态校验
    if (!seat.IsActive)
        return (false, "该座位当前维修中，无法预约");

    // 3. 日期校验
    if (reserveDate <= DateTime.Today.AddDays(-1))
        return (false, "不能预约过去的日期");

    // 4. 当日已过时段拦截
    if (reserveDate == DateTime.Today && IsTimeSlotPast(timeSlot))
        return (false, "该时段已过，请选择其他时段");

    // 5. 时段值域校验
    if (!ValidTimeSlots.Contains(timeSlot))
        return (false, "无效的时段");

    // 6. 冲突检测（数据库唯一索引兜底）
    var conflict = _context.Reservations.Any(r =>
        r.SeatId == seatId &&
        r.ReserveDate == reserveDate &&
        r.TimeSlot == timeSlot &&
        r.Status == "已预约");
    if (conflict)
        return (false, "该时段已被预约，请选择其他时段");

    // 7. 创建预约
    var reservation = new Reservation
    {
        SeatId = seatId,
        UserId = userId,
        ReserveDate = reserveDate,
        TimeSlot = timeSlot,
        Status = "已预约",
        CreatedAt = DateTime.Now
    };
    _context.Reservations.Add(reservation);
    _context.SaveChanges();

    return (true, "预约成功");
}
```

### 2.7 步骤⑥：取消预约

| 维度 | 内容 |
|------|------|
| 页面 | 我的预约页内操作（PC-04） |
| 路由 | `POST /Reservation/Cancel/{id}` |
| Controller | `ReservationController.Cancel(int id)` |
| 权限 | 需要体验账号 + 预约归属当前用户 |
| Service | `IReservationService.CancelReservation(int reservationId, int userId)` |
| DataAccess | `_context.Reservations.FindAsync(id)` → 修改 `Status = "已取消"` → `SaveChangesAsync()` |
| DB 变化 | UPDATE Reservation SET Status = '已取消' WHERE Id = @id |
| 成功路径 | Service 返回 `(true, "取消成功")` → TempData["Success"] → `RedirectToAction("MyReservations")` |
| 失败路径 | Service 返回 `(false, 错误消息)` → TempData["Error"] → `RedirectToAction("MyReservations")` |
| 确认弹窗 | 前端 `confirm()` 弹窗，确认后才发起 POST 请求 |

**Service 核心方法：**

```csharp
public (bool Success, string Message) CancelReservation(int reservationId, int userId)
{
    var reservation = _context.Reservations.Find(reservationId);
    if (reservation == null)
        return (false, "预约记录不存在");

    if (reservation.UserId != userId)
        return (false, "无权取消他人的预约");

    if (reservation.Status != "已预约")
        return (false, "该预约已取消或已完成，无需再次操作");

    // 检查是否已过期（日期+时段双重校验）
    if (reservation.ReserveDate < DateTime.Today)
        return (false, "已过期的预约无法取消");

    if (reservation.ReserveDate == DateTime.Today && IsTimeSlotPast(reservation.TimeSlot))
        return (false, "当前时段已过，无法取消");

    reservation.Status = "已取消";
    _context.SaveChanges();

    return (true, "取消成功");
}

private bool IsTimeSlotPast(string timeSlot)
{
    var now = DateTime.Now;
    return TimeSlotEndHours.TryGetValue(timeSlot, out var endHour)
        && now.Hour >= endHour;
}
```

### 2.8 链路一：状态流转汇总

```
步骤④ 提交预约
  └─ 成功 → Reservation.Status = "已预约" (INSERT)

步骤⑥ 取消预约
  └─ 成功 → Reservation.Status = "已取消" (UPDATE)

页面展示时（步骤⑤）
  └─ 条件判定：
       ReserveDate < Today → 显示"已完成"（不修改 DB）
       ReserveDate == Today && TimeSlot 已过 → 显示"已完成"（不修改 DB）
       其他 → 显示存储在 DB 中的实际状态
```

### 2.9 链路一：异常与边界情况汇总

| 场景 | 触发条件 | 处理方式 | 返回信息 |
|------|----------|----------|----------|
| 座位不存在 | 传入 ID 在 Seat 表中无对应记录 | Service 返回 false | "座位不存在" |
| 座位维修中 | Seat.IsActive = false | Service 返回 false | "该座位当前维修中" |
| 重复预约 | 同一座位+日期+时段已有 Status=已预约 | Service 返回 false + 唯一索引报错 | "该时段已被预约" |
| 过去日期 | ReserveDate < Today | Service 返回 false | "不能预约过去的日期" |
| 过去时段（提交） | reserveDate == Today && 时段已过 | CreateReservation 返回 false | "该时段已过，请选择其他时段" |
| 过去时段（取消） | 当日但时段已过 | CancelReservation 返回 false | "当前时段已过，无法取消" |
| 越权取消 | UserId 不匹配 | Service 返回 false | "无权取消他人的预约" |
| 重复取消 | Status 已是"已取消"或"已完成" | Service 返回 false | "该预约已取消或已完成" |
| 未登录操作 | Session 中无 UserId | Controller 拦截，重定向到首页 | — |
| 无效时段值 | TimeSlot 不是"上午/下午/晚上" | Service 返回 false | "无效的时段" |
| DB 写入失败 | 唯一索引冲突（并发场景） | EF Core 抛异常 → 外层 catch 返回 false | "预约失败，请重试" |

```csharp
public (bool Success, string Message) ValidateLogin(string username, string password)
{
    var admin = _context.Users.FirstOrDefault(u =>
        u.Name == username && u.Role == "Admin");

    if (admin == null)
        return (false, "账号或密码错误");

    if (admin.Password != password)
        return (false, "账号或密码错误");

    return (true, null);
}
```

### 3.8 链路二：权限拦截

```csharp
// AdminBaseController.cs
public class AdminBaseController : Controller
{
    public override void OnActionExecuting(ActionExecutingContext context)
    {
        var role = HttpContext.Session.GetString("UserRole");
        if (role != "Admin")
        {
            context.Result = RedirectToAction("Login");
        }
        base.OnActionExecuting(context);
    }
}

// AdminController.cs

    // 其余 Action 自动继承拦截
}
```

### 3.9 链路二：异常与边界情况汇总

| 场景 | 触发条件 | 处理方式 |
|------|----------|----------|
| 未登录访问管理页 | Session 中无 Admin 角色 | 自动跳转 `/Admin/Login` |
| Session 过期 | IdleTimeout 30 分钟 | 下次请求触发拦截，跳转登录页 |
| 删除有预约的座位 | Seat 表有 Reservation 关联（已预约） | Service 返回 false，提示不可删除 |
| 新增重复座位编号 | SeatNumber 唯一索引冲突 | EF Core 抛异常 → catch 提示"座位编号已存在" |
| 统计页无数据 | Reservation 表为空 | 聚合查询返回 0，页面显示"暂无数据" |
