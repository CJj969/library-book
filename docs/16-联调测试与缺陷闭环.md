前序项目已通过 Sprint 1~3 全部任务卡验收，Sprint 4（功能完善）已交付。本轮作为 **系统级联调、测试与缺陷闭环** 阶段，旨在：

## 测试环境等级与采用方案

| 等级 | 说明 |
|------|------|
| **Level C**（实际采用） | `msedge` 自动化路线被环境阻塞 |

**采用原因：**
- `@playwright/test` 已安装（`npm i -D @playwright/test`）
- `playwright.config.ts` 已配置 `channel: 'msedge'` + `baseURL` + `webServer`
- 本机 **未安装 Microsoft Edge** 和 Google Chrome（仅有 Safari）
- Playwright 无法启动 msedge：`browserType.launch: Chromium distribution 'msedge' is not found`

**未采用 Level A/B 的原因：**
- Level A 需要仓库已有完整 Playwright 骨架 — 本轮从零搭建完成
- Level B 需要本机有 `msedge` — 本机无 Edge/Chrome，Playwright 系统浏览器通道不可用
- 未使用 `npx playwright install` 下载独立 Chromium（按默认要求不下载自带浏览器）

**替代方案：**
1. 脚本烟雾测试（curl 9 项全部通过 ✅）
2. 浏览器手动验证（本机 Safari 可用）

## @playwright/test 安装与 msedge 配置情况

安装命令：
```bash
npm init -y && npm i -D @playwright/test
```

配置文件：`playwright.config.ts`

配置内容：
- `baseURL: 'http://localhost:5258'`
- `webServer: { command: 'dotnet run', port: 5258, reuseExistingServer: true }`
- 两个 project：`msedge`（桌面 1280×720）+ `mobile`（手机 480×812）
- 超时 30s，截图 `only-on-failure`

**Test 脚本：** `e2e/smoke.spec.ts`（9 个用例：用户端 5 + 管理端 3 + 404 页 1）

**执行结果：**
