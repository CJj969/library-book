### 6.3 ❌ 发现的问题

**C04 — 页面树 Controller 映射表遗漏 Admin.CancelReservation（P1）**

页面树 §7.2 的 Controller 映射表（第 296~313 行）列出了 AdminController 的所有 Action，但缺少 `Admin.CancelReservation`（`POST /Admin/Reservations/Cancel/{id}`）。该路由在系统设计 §6 和关键链路 §3.4 中均已确认存在，是页面树映射表的漏列。

**C05 — PRD F08 "标记完成"功能未在实现中出现（P1）**

PRD §6 (F08 预约管理) 要求："每行操作：取消预约（强制取消）、**标记完成**"

但在任何实现文档（系统设计、关键链路）中均未出现"标记完成"操作。关键链路 §3.4 的 Admin 取消预约只做了"取消"。排查原因：需求分析阶段将"已完成"改为查询时逻辑判定（不写入 DB），因此不需要管理员手动"标记完成"。**这是 PRD 未同步更新到实现决策的遗留。建议：在 PRD 中删除"标记完成"的描述，或在 Sprint 3 最终文档更新时一并处理。**

**C06 — 预约管理页筛选条件不完整（P1）**

| 来源 | 筛选条件 |
|------|---------|
| PRD §6 (F08) | 按日期、按区域、按状态（3 个筛选） |
| 关键链路 §3.4 | `string statusFilter, DateTime? dateFilter`（2 个筛选，缺少区域） |
| 任务板 T12-04 验收标准 | "支持按状态/日期筛选"（2 个筛选） |

PRD 要求 3 个筛选维度，但实现文档和任务卡只做了 2 个。"按区域筛选"在预约管理页中被遗漏。建议在 Sprint 2 为预约管理页增加按区域筛选，或在 PRD 中同步删除此要求。
