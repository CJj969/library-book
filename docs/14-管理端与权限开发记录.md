## 核心实现说明

### 权限拦截（T12-01）

```csharp
// AdminBaseController.cs
public class AdminBaseController : Controller
{
    public override void OnActionExecuting(ActionExecutingContext context)
    {
        var role = HttpContext.Session.GetString("UserRole");
        if (role != "Admin")
        {
            context.Result = RedirectToAction("Login");
        }
        base.OnActionExecuting(context);
    }
}
```

- AdminController 继承 AdminBaseController
- Login(GET/POST) 加 `[AllowAnonymous]` 跳过拦截
- 所有未登录请求自动跳转 `/Admin/Login`

### 管理员登录（T12-02 + T12-03）

- `AdminService.ValidateLogin`：查 User 表 `Name == username && Role == "Admin"`，比对 Password
- 成功时写 Session：`AdminId`、`AdminName`、`UserRole="Admin"`
- 失败返回统一提示"账号或密码错误"（不泄露是账号不存在还是密码错误）

### 导航栏切换

_Layout.cshtml 根据 `UserRole` Session 值切换 UI：

- `UserRole == "Admin"`：显示管理员导航（预约管理/座位管理/统计/返回前台）+ 下拉含退出
- 非 Admin：保持原有学生端导航（首页/查座/我的预约）+ 体验账号下拉切换

### 用户端与管理端隔离

| 维度 | 学生 | 管理员 |
|------|------|--------|
| Session 标识 | UserId, UserName, UserRole="Student" | AdminId, AdminName, UserRole="Admin" |
| 导航栏显示 | 首页/查座/我的预约 + 体验账号 | 预约管理/座位管理/统计/返回前台 + 管理员名称 |
| 权限检查 | ReservationController 检查 UserId | AdminBaseController 检查 UserRole=="Admin" |
