# 图书馆座位预约系统 —— 页面树与业务流程

## 1. 页面总数与页面分布

总页面数 **9 页**（8 个业务页面 + 1 个错误页），分布如下：

| 模块 | 页面数 | 页面清单 |
|------|--------|----------|
| 用户端 | 4 页 | 用户首页、座位列表页、座位详情页（含预约表单）、我的预约页 |
| 管理端 | 4 页 | 管理员登录页、预约管理页、座位管理页、统计页 |
| 共享 | 1 页 | 404 错误页 |

> 体验账号切换为全局导航栏下拉框组件，不单独占页。

```
┌─────────────────────────────────────────────────────┐
│                  全局导航栏                          │
│  [Logo/首页]  [查座]  [我的预约]    体验账号: ▼     │
└─────────────────────────────────────────────────────┘

                    ┌──────────┐
                    │ 用户首页  │  ← 系统入口，公告 + 座位概况 + 导航卡片
                    └────┬─────┘
                         │
              ┌──────────┼──────────┐
              │          │          │
        点击"查座"  点击"我的预约"  点击其他导航
              │          │          │
         ┌────▼───┐  ┌──▼──────┐   │
         │座位列表│  │我的预约 │◄──┘
         │  页    │  │   页    │
         └───┬────┘  └─────────┘
             │              ▲
         点击某个座位        │ 预约成功跳转
              │              │
          ┌───▼──────────────┘
          │   座位详情页（含预约表单）
          │   选择时段 → 提交
          │   成功→跳转我的预约
          │   失败→本页显示错误
          └──────────────────┘
```

### 页面跳转关系表

| 当前页 | 操作 | 目标页 | 跳转方式 |
|--------|------|--------|----------|
| 用户首页 | 点击"查座"导航卡片 | 座位列表页 | `GET /Seat/Index` |
| 用户首页 | 点击"我的预约"导航卡片 | 我的预约页 | `GET /Reservation/MyReservations` |
| 座位列表页 | 点击某个座位卡片 | 座位详情页 | `GET /Seat/Detail/{id}` |
| 座位详情页 | 填写预约表单并提交（成功） | 我的预约页 | `POST /Reservation/Create` → 重定向 |
| 座位详情页 | 填写预约表单并提交（失败） | 座位详情页 | 本页显示错误信息，不跳转 |
| 我的预约页 | 点击某条记录的"取消" | 确认弹窗（本页） | 浏览器 confirm 弹窗，无路由跳转 |
| 我的预约页 | 确认取消 → 释放座位 | 我的预约页 | `POST /Reservation/Cancel/{id}` → 本页刷新 |
| 我的预约页 | 导航栏点击"查座" | 座位列表页 | `GET /Seat/Index` |
| 我的预约页 | 导航栏点击"首页" | 用户首页 | `GET /` |
| 座位列表页 | 导航栏点击"我的预约" | 我的预约页 | `GET /Reservation/MyReservations` |
| 全局导航栏 | 切换体验账号 | 当前页（刷新） | 前端切换，页面 reload |

## 4. 用户主链路

```
                                    ┌──────────────────┐
                                    │   用户首页        │
                                    │  (/ 或 /Home/Index)│
                                    └────────┬─────────┘
                                             │
                                     点击"查座"
                                             │
                                    ┌────────▼─────────┐
                                    │   座位列表页       │
                                    │  (/Seat/Index)    │
                                    │  按区域展示座位   │
                                    │  绿色=空闲 红色=已约 灰色=维修  │
                                    └────────┬─────────┘
                                             │
                                     点击空闲座位卡片
                                             │
                                     ┌────────▼──────────────────┐
                                     │   座位详情页（含预约表单） │
                                     │  (/Seat/Detail/5)         │
                                     │  座位信息 + 时段占用时间轴 │
                                     │  ─── 预约表单 ───        │
                                     │  选择日期                 │
                                     │  选择时段（上午/下午/晚上）│

                                     │  成功→跳转我的预约        │

                                         提交成功
                                              │
                                     ┌────────▼─────────┐

                                    │  (/Reservation/   │
                                    │   MyReservations) │
                                    │  显示新增的预约    │
                                    │  状态: 已预约(蓝)  │
                                    └────────┬─────────┘
                                             │
                                     点击"取消预约"
                                             │
                                    ┌────────▼─────────┐
                                    │   确认弹窗         │
                                    │   "确定取消？"     │
                                    │   [确认] [取消]    │

                                             │
                                       点击"确认"
                                             │
                                    ┌────────▼─────────┐

                                    │  状态: 已取消(灰)  │

        ↑←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

## 6. 状态流转

### 6.1 预约状态

```
                    ┌─────────┐
                    │ 已预约   │  ← 预约提交成功
                    │ (Active) │
                    └────┬────┘
                         │
               ┌─────────┼─────────┐
               │                   │
           用户/管理员           预约日期已过
           手动取消              （系统自动判定）
               │                   │
          ┌────▼────┐        ┌────▼────┐
          │ 已取消   │        │ 已完成   │
          │(Cancelled)│        │(Completed)│
          └──────────┘        └──────────┘
```

**各状态页面表现：**

| 状态 | 我的预约页显示 | 座位列表页显示 |
|------|---------------|---------------|
| 已预约 (Active) | 蓝色标签，"取消预约"按钮可点击 | 该时段对应座位显示为红色（已预约） |
| 已取消 (Cancelled) | 灰色标签，无操作按钮 | 该时段座位恢复为绿色（空闲） |
| 已完成 (Completed) | 绿色标签，无操作按钮 | 该时段座位显示为绿色（空闲，可预约新的） |

### 6.2 座位状态

```
     ┌───────────┐
     │  空闲      │  ← 初始状态 / 维修完成 / 预约取消/到期后释放
     │ (Available)│
     └─────┬─────┘
           │
     管理员在后台切换
           │
     ┌─────▼───────┐
     │  维修中      │
     │ (Maintenance)│
     └──────────────┘
```

> 座位表中不存"已预约"状态。判断座位是否被占 = 查询 Reservation 表中是否存在该 SeatId + 当天 + 时段重叠 + Status=Active 的记录。

**各状态页面表现：**

| 状态 | 座位列表页显示 | 座位详情页操作 |
|------|---------------|---------------|
| 空闲 | 绿色卡片，可点击 | "预约此座位"按钮可点击 |
| 已预约 | 红色卡片，不可点击 | 预约按钮禁用，提示"已被预约" |
| 维修中 | 灰色卡片，不可点击 | 预约按钮禁用，提示"维修中" |

## 7. 页面与流程关系说明

### 7.1 页面职责矩阵

| 页面 | 用户角色 | 核心操作 | 依赖数据 |
|------|----------|----------|----------|
| 用户首页 | 学生 | 查看概况、选择去向 | Seat（聚合统计）、体验账号 |

| 统计页 | 管理员 | 查看统计数据 | Reservation、Seat（聚合查询） |
